# 审核流程调整总结

## 调整完成 ✅

已成功调整审核流程，现在审核通过后的记录会直接进入放行队列，由放行工程师执行放行操作。

## 主要变更

### 1. 队列划分调整
- **审核队列**：只显示 `submitted` 状态（待审核）
- **放行队列**：显示 `peer_checked` 和 `rii_approved` 状态（待放行）

### 2. 状态标签统一
- `peer_checked` 状态标签从"待复核"改为"待放行"
- 所有前端页面保持一致

### 3. 修改的文件
```
backend/src/services/maintenanceService.js
frontend/src/pages/ApprovalWorkbenchPage.vue
frontend/src/pages/RecordsCenterPage.vue
docs/审核流程调整说明.md (新增)
```

## 业务流程

### 标准流程
```
提交 → 审核通过 → 放行 → 完成
(submitted) → (peer_checked) → (released)
```

### RII流程
```
提交 → 审核通过 → RII批准 → 放行 → 完成
(submitted) → (peer_checked) → (rii_approved) → (released)
```

## 测试步骤

1. 启动系统：`.\startall.ps1`
2. 登录审核工程师账号（E2001）
3. 在审核队列中对 `submitted` 记录执行"审核通过"
4. 确认记录从审核队列消失，出现在放行队列
5. 登录放行工程师账号（E2002）
6. 在放行队列中对 `peer_checked` 记录执行"放行"
7. 确认记录状态变为 `released`
