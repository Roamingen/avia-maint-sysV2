# 审核流程调整说明

## 调整日期
2026-03-09

## 调整内容

### 问题描述
原系统中，审核通过后记录进入 `peer_checked` 状态，但仍然显示在"审核队列"中，标签显示为"待复核"，容易造成混淆。实际上审核通过后应该直接进入放行队列，由放行工程师执行最终放行操作。

### 调整方案

#### 1. 状态流转逻辑
- **提交后**: `submitted` (待审核)
- **审核通过后**: `peer_checked` (待放行) - 进入放行队列
- **RII批准后**: `rii_approved` (待放行) - 进入放行队列
- **最终放行**: `released` (已放行)

#### 2. 工作台队列划分

**审核队列**：
- 只显示 `submitted` 状态的记录
- 审核工程师可以对这些记录执行"审核通过"或"驳回"操作

**放行队列**：
- 显示 `peer_checked` 和 `rii_approved` 状态的记录
- 放行工程师对这些记录执行"放行"操作

#### 3. 状态标签统一

| 数据库状态 | 前端显示标签 | 说明 |
|-----------|------------|------|
| `submitted` | 待审核 | 刚提交，等待审核工程师审核 |
| `peer_checked` | 待放行 | 审核通过，等待放行工程师放行 |
| `rii_approved` | 待放行 | RII批准，等待放行工程师放行 |
| `released` | 已放行 | 最终放行完成 |
| `rejected` | 已驳回 | 被驳回，需要重新提交 |
| `revoked` | 已作废 | 已作废 |
| `draft` | 草稿 | 草稿状态 |

## 业务流程示例

### 非RII工作流程
```
提交 (submitted)
  ↓ 审核工程师审核通过
待放行 (peer_checked)
  ↓ 放行工程师放行
已放行 (released)
```

### RII工作流程
```
提交 (submitted)
  ↓ 审核工程师审核通过
待放行 (peer_checked)
  ↓ RII检查员批准
待放行 (rii_approved)
  ↓ 放行工程师放行
已放行 (released)
```

## 角色权限说明

### 审核工程师 (Reviewer)
- 权限：`record.approve`
- 可执行操作：
  - 对 `submitted` 状态的记录执行"审核通过" (`reviewer_sign`)
  - 对 `submitted` 状态的记录执行"驳回" (`reject`)

### RII检查员 (RII Inspector)
- 权限：`record.approve`
- 可执行操作：
  - 对 `peer_checked` 状态的RII工作执行"RII批准" (`rii_approve`)

### 放行工程师 (Release Authority)
- 权限：`record.approve`
- 可执行操作：
  - 对 `peer_checked` 或 `rii_approved` 状态的记录执行"放行" (`release`)

## 测试建议

### 测试场景1：非RII工作审核放行
1. 提交工程师提交一条非RII检修记录
2. 审核工程师在审核队列中看到该记录（状态：待审核）
3. 审核工程师点击"审核通过"
4. 记录从审核队列消失，出现在放行队列（状态：待放行）
5. 放行工程师在放行队列中看到该记录
6. 放行工程师点击"放行"
7. 记录状态变为"已放行"

### 测试场景2：RII工作审核放行
1. 提交工程师提交一条RII检修记录（`isRII: true`）
2. 审核工程师审核通过
3. 记录出现在放行队列（状态：待放行，实际为 `peer_checked`）
4. RII检查员执行"RII批准"
5. 记录仍在放行队列（状态：待放行，实际为 `rii_approved`）
6. 放行工程师执行"放行"
7. 记录状态变为"已放行"

### 测试场景3：驳回流程
1. 审核工程师在审核队列中对记录点击"驳回"
2. 输入驳回原因
3. 记录状态变为"已驳回"
4. 提交工程师修改后重新提交新版本（revision + 1）
5. 新版本重新进入审核队列

## 注意事项

1. **状态标签一致性**：所有前端页面的状态标签必须保持一致
2. **权限控制**：确保只有具有相应权限的用户才能执行对应操作
3. **智能合约验证**：所有状态变更都会在智能合约层面进行验证
4. **指定签名人**：如果配置了指定签名人，只有指定的人员才能执行签名操作
